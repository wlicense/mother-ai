## 📝 初めての利用（アカウント作成）

### 新規アカウントを作成する方法

**ブラウザで:** http://localhost:3347

1. **「利用申請」をクリック**
2. **情報を入力:**
   ```
   名前: あなたの名前（何でもOK）
   メール: test@example.com（何でもOK）
   パスワード: test1234（何でもOK）
   利用目的: テスト利用
   ```
3. **「申請する」をクリック**

**モックモードでは自動承認されます！**

## 🚀 ログイン手順

### Step 1: アカウント作成後

1. **「ログイン」をクリック**
2. **作成したメール・パスワードを入力:**
   ```
   メール: test@example.com
   パスワード: test1234
   ```
3. **「ログイン」をクリック**

### Step 2: ダッシュボード表示

ログイン成功すると:
- ✅ ダッシュボードが表示される
- ✅ 「新規プロジェクト」ボタンが見える
- ✅ ユーザー名が右上に表示される

## ⚠️ トラブルシューティング

### 問題1: ログインできない

**原因**: アカウントがまだ作成されていない

**解決策**:
1. 「利用申請」から新規アカウント作成
2. メール・パスワードを正確に入力

---

### 問題2: 「審査中」と表示される

**原因**: 承認待ち状態（通常は自動承認）

**解決策**:
```bash
# 管理者として承認する
curl -X POST http://localhost:8572/api/v1/admin/applications/{id}/approve
```

または:
- http://localhost:3347/admin/applications にアクセス
- 申請を手動承認

---

### 問題3: パスワードを忘れた

**解決策**: 新しいアカウントを作成してください

モックモードでは、パスワードリセット機能は不要です。
新しいメールアドレスで別アカウントを作成できます。

## 📊 アカウント種別

### ユーザーアカウント（デフォルト）

**できること**:
- ✅ プロジェクト作成
- ✅ Phase 1-14実行
- ✅ コード生成
- ✅ デプロイ手順取得

**できないこと**:
- ❌ 他ユーザーの管理
- ❌ 申請の承認/却下
- ❌ API使用量の閲覧

---

### 管理者アカウント

**追加でできること**:
- ✅ 全ユーザーの管理
- ✅ 申請の承認/却下
- ✅ API使用量ダッシュボード
- ✅ システム設定変更

**作成方法**:
```bash
# データベースで直接role='admin'に変更
UPDATE users SET role='admin' WHERE email='test@example.com';
```

## 🔐 セキュリティ情報

### モックモードでは

- ✅ どんなパスワードでもOK
- ✅ 実在しないメールアドレスでOK
- ✅ データはローカルDBのみ
- ✅ 外部には送信されない

### 本番環境では

- ⚠️ 強力なパスワード推奨
- ⚠️ 実在するメールアドレス必要
- ⚠️ JWT認証で保護
- ⚠️ パスワードはbcryptでハッシュ化
