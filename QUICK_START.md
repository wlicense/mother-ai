## 📝 操作手順（5ステップ）

### Step 1: ブラウザでアクセス

```bash
# ブラウザで開く
open http://localhost:3347
```

または手動で `http://localhost:3347` をブラウザに入力

---

### Step 2: アカウント作成

1. トップページの「利用申請」をクリック
2. 以下の情報を入力:
   ```
   名前: テストユーザー
   メール: test@example.com
   パスワード: test1234
   利用目的: テスト（何でもOK）
   ```
3. 「申請する」をクリック

**注意**: モックモードでは自動承認されます

---

### Step 3: ログイン

1. 「ログイン」をクリック
2. 入力:
   ```
   メール: test@example.com
   パスワード: test1234
   ```
3. 「ログイン」をクリック

---

### Step 4: プロジェクト作成

1. ダッシュボードで「新規プロジェクト」をクリック
2. 入力:
   ```
   プロジェクト名: ToDoアプリ
   説明: シンプルなタスク管理アプリを作りたい
   ```
3. 「作成」をクリック

---

### Step 5: Phase 1-14 を試す

プロジェクト詳細画面で、14個のPhaseカードが表示されます：

#### **Phase 1: 要件定義エージェント**
1. Phase 1カードをクリック
2. チャット画面で質問:
   ```
   ToDoアプリに必要な機能を教えてください
   ```
3. AIが要件を提案（モックレスポンス）

#### **Phase 2: コード生成エージェント**
1. Phase 1完了後、Phase 2が解放される
2. Phase 2をクリック
3. 「コード生成開始」をクリック
4. 27個のファイルが生成される（モック）

#### **Phase 3: デプロイエージェント**
1. Phase 2完了後、Phase 3が解放される
2. デプロイ設定を入力（モック）
3. Vercel + Cloud Runへのデプロイ手順が表示される

#### **Phase 4-14: 拡張エージェント**
- Phase 4: 自己改善（マザーAI専用）
- Phase 5: テスト生成
- Phase 6: ドキュメント生成
- Phase 7: デバッグ支援
- Phase 8: パフォーマンス最適化
- Phase 9: セキュリティ監査
- Phase 10: データベース設計
- Phase 11: API設計
- Phase 12: UX/UI改善
- Phase 13: リファクタリング

```bash

cd backend
source venv/bin/activate
export USE_REAL_AI=false
python -m app.main

# フロントエンド（別ターミナル）
cd frontend
npm run dev
```
